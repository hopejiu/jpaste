## WebDAV 同步方案设计 (坚果云)

15:17 — grill-with-docs 质询了 8 个设计决策后定型。

### 决策汇总
1. **双向合并** — text相同取最新 `updated_at`，`content_hash` 做 identity key
2. **Per-hash JSON 文件** — WebDAV 上 `entries/{hash前缀2字符}/{hash}.json`，实现真增量同步
3. **无 manifest** — 用本地 SQLite 的 `updated_at` 直接对比 PROPFIND 结果
4. **Push** — 每次剪贴板变化立即 PUT；失败指数退避（1min→2min→4min→…）
5. **Pull** — 每 60 秒 PROPFIND + 逐条对比；失败不影响周期
6. **Settings sync** — push 立即推、pull 仅启动时一次
7. **Delete** — 使用共享的 `retain_days`（settings sync 保证一致），push 时清理云端过期文件
8. **离线** — 静默退避 + 同步状态指示器（绿/灰/黄/红）

## WebDAV 同步实现

16:00 — 完成 sync 功能实现，`wails3 build` 通过。

### Go 后端 (internal/sync/)
- `config.go` — WebDAV 凭证 `{url, username, password}` 读写 `webdav.json`
- `webdav.go` — HTTP 客户端：PROPFIND/GET/PUT/DELETE/MKCOL + Basic Auth，XML 解析
- `service.go` — Sync Service: push/pull/backoff/cleanup 循环、状态管理
  - Push: `PushEntry(ContentHash, Content)` → 计算 SHA256 → PUT `entries/{prefix}/{hash}.json`
  - Pull: 每 60s PROPFIND → 逐条对比 SQLite `updated_at` → 拉取新/更新的条目
  - Backoff: 指数退避 1m→2m→4m→8m→16m→32m
  - Cleanup: 推送完成后删除云端超过 `retain_days` 的条目
  - Settings: `PushSettings` 在每次 `OnSettingsChange` 触发，`PullSettings` 启动时执行
